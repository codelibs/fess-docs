================
製品サポート期限
================

製品サポート期限を超えた以下のプロダクトは、今後、メンテナンス等の更新がありません。
CodeLibs Projectでは、サポート対象のリリースに移行することを強くお勧めします。
これにより、必要なサービスやサポートが利用できなくなっているという事態を回避することができます。
最新のリリースは、`ダウンロードページ <downloads.html>`__ からダウンロードできます。

製品サポート期限を超えた製品でサポートが必要な場合は、`商用サポート <https://www.n2sm.net/products/n2search.html>`__ にご相談ください。

.. warning::

   **サポート終了前の推奨アクション**

   サポート終了日の前に、以下のアクションを計画的に実施してください：

   1. **バックアップの取得**: 設定ファイルとインデックスデータをバックアップ
   2. **テスト環境での検証**: 本番移行前に新バージョンで動作確認
   3. **リリースノートの確認**: 破壊的変更や非推奨機能の確認
   4. **移行スケジュールの策定**: ダウンタイムを考慮した計画策定

アップグレードパス
================

現在のバージョンから最新版へのアップグレードパスは以下の通りです。

.. list-table::
   :header-rows: 1
   :widths: 25 25 50

   * - 現在のバージョン
     - 推奨経路
     - 備考
   * - 15.x → 15.5
     - 直接アップグレード可能
     - `アップグレード手順 <15.5/install/upgrade.html>`__ を参照
   * - 14.x → 15.5
     - 直接アップグレード可能
     - 設定ファイルの変更点に注意
   * - 13.x → 15.5
     - 14.x 経由を推奨
     - 13.x → 14.19 → 15.5 の順でアップグレード
   * - 12.x 以前 → 15.5
     - 段階的アップグレード必須
     - 1〜2メジャーバージョンずつアップグレード

.. note::

   詳細なアップグレード手順は `アップグレードガイド <15.5/install/upgrade.html>`__ を参照してください。
   大規模な環境や複雑な構成の場合は、`商用サポート <support-services.html>`__ へのご相談を推奨します。

よくある質問
==========

Q: サポート終了後も Fess を使い続けることはできますか？
----------------------------------------------------

A: 技術的には使用可能ですが、セキュリティ修正やバグ修正は提供されません。
セキュリティリスクを軽減するため、サポート対象バージョンへのアップグレードを強く推奨します。

Q: アップグレードにはどのくらいの時間がかかりますか？
-------------------------------------------------

A: 環境の規模によりますが、一般的には2〜4時間程度です。
大規模環境や複雑な構成の場合は、事前にテスト環境での検証を推奨します。
詳細は `アップグレード手順 <15.5/install/upgrade.html>`__ を参照してください。

Q: サポート終了バージョンで問題が発生した場合はどうすればよいですか？
----------------------------------------------------------------

A: 以下の選択肢があります：

1. **最新バージョンへのアップグレード**: 推奨される対応
2. **コミュニティフォーラムでの質問**: 他のユーザーからアドバイスを得られる場合があります
3. **商用サポートへの相談**: `N2SM の商用サポート <support-services.html>`__ では、特定バージョンの保守も対応可能です

Q: アップグレード前に確認すべきことは何ですか？
--------------------------------------------

A: 以下を確認してください：

1. `リリースノート <https://github.com/codelibs/fess/releases>`__ で破壊的変更を確認
2. OpenSearch のバージョン互換性を確認
3. カスタマイズしている場合は、設定やプラグインの互換性を確認
4. 十分なバックアップを取得

Q: Docker 環境でのアップグレードは特別な手順が必要ですか？
------------------------------------------------------

A: Docker ボリュームのバックアップと新しい Docker Compose ファイルの取得が必要です。
詳細は `Docker インストールガイド <15.5/install/install-docker.html>`__ を参照してください。
