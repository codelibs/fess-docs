==========
AI搜索模式
==========

概述
====

使用 |Fess| 的AI搜索模式功能（RAG聊天），除了传统的关键词搜索之外，
还可以通过自然对话的形式查找信息。输入问题后，AI助手会检索相关文档，
并基于文档内容生成易于理解的回答。

.. note::
   AI搜索模式在初始状态下处于禁用状态。若要使用该功能，需要系统管理员先启用
   该功能，并配置好LLM（大语言模型）提供商。配置方法请参阅
   :doc:`../config/rag-chat` 和 :doc:`../config/llm-overview`。

AI搜索模式的特点
================

对话形式的搜索
--------------

无需构思关键词，就可以像日常对话一样提问。

示例:

- 「请告诉我Fess的安装方法」
- 「应该如何爬取文件？」
- 「搜索结果不显示时该如何处理？」

理解上下文的回答
----------------

AI助手会分析问题的意图，检索相关文档，并从多个搜索结果中提取
所需信息，以整理好的形式进行回答。

明示来源
--------

回答的下方会以“来源”的形式，列出作为回答依据的文档。每个来源都是
带编号的链接，点击即可直接查看原始文档。此外，回答正文中有时会以
``[1]``、``[2]`` 之类的编号显示引用，这些编号与来源列表中的编号
相对应。

延续对话
--------

提问不止一次即结束，还可以继续对话。
AI助手会结合之前问题和回答的上下文，回答追加的问题。

聊天搜索的使用方法
==================

打开AI搜索模式
--------------

1. 打开 |Fess| 的搜索画面
2. 点击画面顶部导航栏中的“AI搜索”链接（机器人图标）
3. 显示AI搜索模式的画面

.. note::
   “AI搜索”链接仅在AI搜索模式已启用且LLM提供商可用时才会显示。
   如果链接未显示，请参阅`常见问题`_。

输入问题
--------

1. 在画面下方的文本框中输入问题（每次提问最多4000个字符）
2. 点击“发送消息”按钮（纸飞机图标），或按 Enter 键
3. AI助手开始生成回答

.. note::
   如果需要换行，请按 Shift+Enter 键代替 Enter 键。

.. note::
   生成回答可能需要几秒到十几秒的时间。处理过程中，会以进度显示
   （分析 → 搜索 → 评估 → 获取 → 回答）的形式展示当前处理阶段，
   并显示类似“思考中...”、“搜索中...”、“正在检查搜索结果...”、
   “正在获取文档...”、“正在生成回答...”的消息。搜索完成后，
   还会显示找到的文档数量。

确认回答
--------

系统会显示AI助手的回答。回答中包含以下信息:

- **回答正文**: 针对问题的回答（以Markdown格式排版）
- **来源**: 作为回答依据的文档链接列表（带编号的链接，点击后会在新标签页中打开原始文档）

回答正文中可能包含 ``[1]``、``[2]`` 之类的引用编号，这些编号与来源列表中的编号相对应。
每个回答都带有复制按钮，可以将回答内容复制到剪贴板。

.. note::
   AI仅会将其判断为相关性较高的靠前文档用作回答依据。因此，
   “来源”的数量可能会少于搜索命中的文档数量。

缩小搜索范围
------------

根据主题不同，可以通过画面上方的“筛选”按钮，使用标签、文件类型、更新时间、
大小等条件来缩小搜索范围。当只想针对特定范围内的文档向AI提问时，
这个功能会很有用。

继续对话
--------

如果还有其他问题，可以直接继续提问:

- 「请再详细说明一下」
- 「还有其他方法吗？」
- 「关于〇〇的详细说明」

AI助手会结合之前的对话上下文进行回答。

开始新对话
----------

如果想询问其他主题，请点击“新对话”按钮（+图标）。
这样会清除之前的对话历史，从而开始新的对话。

有效提问的技巧
==============

具体提问
--------

相比模糊的问题，具体的问题更容易得到恰当的回答。

.. list-table::
   :header-rows: 1
   :widths: 50 50

   * - 模糊的问题
     - 具体的问题
   * - 设置方法是什么？
     - 如何更改Fess的内存设置？
   * - 出现错误
     - 搜索时出现“找不到索引”的错误
   * - 关于爬取
     - 爬取网站时应该如何设置排除规则？

包含背景信息
------------

包含具体情况和目的，可以获得更恰当的回答。

好的示例:

- 「我在Docker环境中运行Fess。想更改日志的保存位置，该怎么做？」
- 「我是第一次使用Fess。首先应该做什么？」

分步提问
--------

复杂的问题通过分步提问会更容易理解。

1. 「Fess可以爬取文件共享吗？」
2. 「请告诉我SMB协议的连接方法」
3. 「如果是需要认证的共享文件夹，该怎么办？」

常见问题
========

Q: 未显示AI搜索模式
-------------------

A: AI搜索模式可能尚未启用。
“AI搜索”链接仅在功能已启用（``rag.chat.enabled=true``）且LLM提供商
（OpenAI、Gemini、Ollama等）已配置并可用时才会显示。
请向系统管理员确认AI搜索模式是否已启用，以及LLM提供商是否配置正确
（详情请参阅 :doc:`../config/rag-chat`）。

Q: 需要一段时间才能显示回答
---------------------------

A: 由于AI需要先分析问题，再对文档进行搜索和评估，然后才生成回答，
因此可能需要几秒到十几秒的时间。处理过程中，会显示当前的处理阶段
（“思考中...”、“搜索中...”、“正在检查搜索结果...”、“正在获取文档...”、
“正在生成回答...”）。

Q: “来源”看起来不正确
---------------------

A: 请点击“来源”链接，确认原始文档。
AI是基于搜索结果生成回答的，但解释内容有时可能存在误差。
建议对重要信息务必在原始文档中进行确认。

Q: 之前的对话似乎被遗忘了
-------------------------

A: 对话会话可能已超时。
如果在一定时间内（默认为30分钟）没有任何操作，对话历史就会被清除。
此外，对话历史仅临时保存在服务器的内存中，因此服务器重启后也会丢失。
这种情况下，请开始新的对话。

.. note::
   这里所说的“会话”是指AI搜索模式的对话会话，与 |Fess| 的登录会话是两回事。

Q: 针对特定问题无法获得回答
---------------------------

A: 可能存在以下原因:

- 搜索目标文档中没有相关信息
- 问题过于模糊，无法进行恰当的搜索
- 已达到速率限制（超过每分钟请求数上限，或超过并发数上限）

请换一种方式表述问题，或稍等片刻后重试。

Q: 可输入的字符数有限制吗？
---------------------------

A: 每次提问最多可输入4000个字符。文本框下方会显示字符数计数器，
接近上限时会出现警告提示。如果问题较长，请提炼要点并进行精简。

Q: 可以用日语以外的语言提问吗？
-------------------------------

A: 可以，大多数情况下，也可以使用英语或其他语言提问。
AI会根据浏览器或画面的显示语言，尽可能使用同一种语言进行回答。
不过，这只是尽力而为的处理方式，根据具体情况，有时可能无法得到
预期语言的回答。

注意事项
========

关于AI的回答
------------

- AI的回答是基于搜索结果生成的
- 不保证回答内容的准确性
- 进行重要判断时，请务必确认原始文档（来源）
- 最新信息请以官方文档为准进行确认

关于隐私
--------

- 输入的问题会用于搜索，以及由所配置的LLM提供商进行AI处理
- 如果配置为使用外部LLM服务（如OpenAI、Gemini等），问题内容和搜索结果会被发送到该服务。
  如果希望仅在公司内部进行处理，请与管理员协商是否可以使用在本地运行的提供商（如Ollama等）
- 对话历史会临时保存在服务器内存中，会话超时、执行“新对话”或服务器重启都会将其删除
- 与普通搜索一样，系统会应用基于角色（权限）和标签的访问控制，无法查看的文档不会成为回答的对象
- 根据系统设置，可能会记录日志
