=================
검색 필드 지정 검색
=================

필드 지정 검색
==============

|Fess| 에서 크롤링한 결과는 제목이나 본문 등 필드별로 저장됩니다. 이러한 필드를 지정하여 검색하면 문서 유형별이나 크기별 등 세밀한 검색 조건을 지정할 수 있습니다.

사용 가능한 필드
---------------

기본적으로 다음 필드를 지정하여 검색할 수 있습니다.

.. list-table::
   :header-rows: 1

   * - 필드명
     - 설명
     - 데이터 타입
   * - url
     - 크롤링한 URL
     - 키워드
   * - host
     - 크롤링한 URL에 포함된 호스트명
     - 키워드
   * - site
     - URL에서 스킴과 쿼리 문자열을 제외한 문자열(호스트명과 경로). 전방 일치로 검색됩니다
     - 키워드
   * - title
     - 제목
     - 텍스트
   * - content
     - 본문
     - 텍스트
   * - content_length
     - 문서 크기(바이트)
     - 숫자
   * - last_modified
     - 문서의 최종 수정 일시
     - 일시
   * - timestamp
     - 문서의 등록 일시(최종 수정 일시를 가져올 수 없는 경우 크롤링 실행 시각)
     - 일시
   * - mimetype
     - 문서의 MIME 타입(예: ``text/html``)
     - 키워드
   * - filetype
     - MIME 타입으로 결정되는 파일 종류(예: ``html``, ``word``, ``pdf``. 해당하지 않는 경우 ``others``)
     - 키워드
   * - filename
     - URL 경로 끝의 파일명
     - 키워드
   * - label
     - 문서에 부여된 레이블 값
     - 키워드
   * - lang
     - 문서의 언어 코드(예: ``ja``, ``en``)
     - 키워드
   * - anchor
     - HTML 페이지 내에서 추출한 링크 대상 URL(웹 크롤링 시에만)
     - 키워드
   * - click_count
     - 문서가 클릭된 횟수
     - 숫자
   * - favorite_count
     - 문서가 즐겨찾기에 등록된 횟수
     - 숫자

표: 사용 가능한 필드 목록

"데이터 타입"은 필드별 검색 방법의 차이를 나타냅니다.

* 키워드: 값 전체를 대상으로 완전 일치로 검색합니다. 와일드카드 검색이나 프리픽스(전방 일치) 검색과 조합할 수도 있습니다.
* 텍스트: 형태소 분석 등으로 분할된 단어를 대상으로 전문 검색을 합니다. title이나 content가 이에 해당합니다.
* 숫자·일시: :doc:`범위 지정 검색 <search-range>` 을 이용할 수 있습니다.

필드를 지정하지 않은 경우는 title 및 content를 대상으로 검색합니다. 설정에 따라 검색 대상 필드를 추가할 수도 있습니다.

.. note::
    크롤링 대상에 따라 값이 등록되지 않는 필드도 있습니다. 예를 들어 anchor는 웹 크롤링 시에만, lang은 HTML에 언어 속성이 있는 경우에만 등록됩니다. 또한 segment(크롤링 실행 단위를 나타내는 세션 ID)나 doc_id(시스템이 부여하는 내부 ID) 등의 필드도 지정할 수 있지만, 일반적인 검색에서는 사용하지 않습니다.

HTML 파일을 검색 대상으로 하는 경우 title 태그가 title 필드에, body 태그 이하의 문자열이 content 필드에 등록됩니다.

사용 방법
------

필드 지정 검색을 하는 경우는 "필드명:검색어"와 같이 필드명과 검색어를 콜론(:)으로 구분하여 검색 양식에 입력합니다.

title 필드에 대해 fess를 검색어로 검색하는 경우는 다음과 같이 입력합니다.

::

    title:fess

위 검색으로 title 필드에 fess가 포함된 문서가 검색 결과로 표시됩니다.

url 필드에 대해 검색하는 경우는 다음과 같이 입력합니다.

::

   url:https\:\/\/fess.codelibs.org\/ja\/15.8\/*
   url:"https://fess.codelibs.org/ja/15.8/*"

전자는 와일드카드 검색을 사용할 수 있으므로 ``url:*\/\/fess.codelibs.org\/*`` 와 같은 작성도 가능합니다. url에 포함된 ":"나 "/"는 검색 쿼리의 특수 문자이므로 ``\`` 를 붙여 이스케이프하십시오(:doc:`special-char` 참조). 후자는 와일드카드 검색을 사용할 수 없지만, 끝을 ``*`` 로 한 단어는 프리픽스(전방 일치) 검색으로 처리됩니다.

.. tip::
    URL의 일부로 검색하고 싶은 경우는 ``inurl:`` 을 사용하면 URL에 대한 부분 일치 검색을 할 수 있습니다. 예를 들어 ``inurl:15.8`` 은 URL에 ``15.8`` 을 포함하는 문서를 검색합니다.

mimetype나 filetype, site와 같이 값에 "/"를 포함하는 필드를 지정하는 경우에는 ``mimetype:"text/html"`` 처럼 인용부호로 감싸거나, ``mimetype:text\/html`` 처럼 이스케이프하십시오.

관련 검색 구문에 대해서는 :doc:`와일드카드 검색 <search-wildcard>`, :doc:`범위 지정 검색 <search-range>`, :doc:`정렬 검색 <search-sort>`, :doc:`특수 문자 <special-char>` 도 참조하십시오.
