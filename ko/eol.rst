================
제품 지원 기한
================

제품 지원 기한을 초과한 제품은 앞으로 유지보수 등의 업데이트가 없습니다.
CodeLibs Project에서는 지원 대상 릴리스로 이전할 것을 강력히 권장합니다.
이를 통해 필요한 서비스와 지원을 이용할 수 없게 되는 사태를 회피할 수 있습니다.
최신 릴리스는 `다운로드 페이지 <downloads.html>`__ 에서 다운로드할 수 있습니다.

제품 지원 기한을 초과한 제품에서 지원이 필요한 경우 `상용 지원 <https://www.n2sm.net/products/n2search.html>`__ 으로 문의하시기 바랍니다.

.. warning::

   **지원 종료 전 권장 조치**

   지원 종료일 이전에 다음 조치를 계획하고 실행하세요:

   1. **백업 작성**: 설정 파일 및 인덱스 데이터를 백업합니다
   2. **스테이징 환경에서 테스트**: 프로덕션 마이그레이션 전에 새 버전에서 동작을 확인합니다
   3. **릴리스 노트 확인**: 호환되지 않는 변경 사항 및 폐지된 기능을 확인합니다
   4. **마이그레이션 일정 계획**: 다운타임 요구 사항을 고려한 계획을 수립합니다

업그레이드 경로
================

다음 표는 현재 버전에서 최신 릴리스로의 권장 업그레이드 경로를 보여줍니다.

.. list-table::
   :header-rows: 1
   :widths: 25 25 50

   * - 현재 버전
     - 권장 경로
     - 비고
   * - 15.x에서 15.5로
     - 직접 업그레이드 가능
     - `업그레이드 가이드 <15.5/install/upgrade.html>`__ 참조
   * - 14.x에서 15.5로
     - 직접 업그레이드 가능
     - 설정 파일의 변경 사항에 주의
   * - 13.x에서 15.5로
     - 14.x 경유 권장
     - 순서대로 업그레이드: 13.x에서 14.19로, 14.19에서 15.5로
   * - 12.x 이전에서 15.5로
     - 단계적 업그레이드 필요
     - 한 번에 1~2개 메이저 버전씩 업그레이드

.. note::

   자세한 업그레이드 절차는 `업그레이드 가이드 <15.5/install/upgrade.html>`__ 를 참조하세요.
   대규모 환경이나 복잡한 구성의 경우 `상용 지원 <support-services.html>`__ 을 이용하는 것을 권장합니다.

======================

- `릴리스 노트 <https://github.com/codelibs/fess/releases>`__ - 각 버전의 변경 사항 및 주의 사항

- `Docker 업그레이드 <15.5/install/install-docker.html>`__ - Docker 환경에서의 업그레이드

자주 묻는 질문
==========================

Q: 지원 기한이 지난 후에도 Fess를 계속 사용할 수 있나요?
------------------------------------------------------------

A: 기술적으로는 가능하지만 보안 수정 및 버그 수정이 제공되지 않습니다.
보안 리스크를 줄이기 위해 지원 대상 버전으로 업그레이드할 것을 강력히 권장합니다.

Q: 업그레이드에 얼마나 시간이 걸리나요?
------------------------------------------

A: 환경 규모에 따라 다르지만 일반적으로 2~4시간 정도입니다.
대규모 환경이나 복잡한 구성의 경우 먼저 스테이징 환경에서 테스트할 것을 권장합니다.
자세한 내용은 `업그레이드 가이드 <15.5/install/upgrade.html>`__ 를 참조하세요.

Q: 지원 종료 버전에서 문제가 발생하면 어떻게 해야 하나요?
--------------------------------------------------------------------------

A: 다음 옵션이 있습니다:

1. **최신 버전으로 업그레이드**: 권장 조치
2. **커뮤니티 포럼에서 질문**: 다른 사용자로부터 조언을 받을 수 있습니다
3. **상용 지원에 문의**: `N2SM 상용 지원 <support-services.html>`__ 에서 특정 버전의 유지보수를 제공받을 수 있습니다

Q: 업그레이드 전에 무엇을 확인해야 하나요?
--------------------------------------------

A: 다음 사항을 확인해 주세요:

1. `릴리스 노트 <https://github.com/codelibs/fess/releases>`__ 에서 호환되지 않는 변경 사항 확인
2. OpenSearch 버전 호환성 확인
3. 커스터마이즈가 있는 경우 설정 및 플러그인 호환성 확인
4. 철저한 백업 작성

Q: Docker 환경에서의 업그레이드에 특별한 절차가 필요한가요?
-----------------------------------------------------------------

A: Docker 볼륨을 백업하고 새로운 Docker Compose 파일을 가져와야 합니다.
자세한 내용은 `Docker 설치 가이드 <15.5/install/install-docker.html>`__ 를 참조하세요.
